# 遊戲筆記網站專案文件

## 1. 專案背景

遊戲玩家在遊玩過程中，經常需要記錄任務進度、角色養成、裝備配置、地圖位置、攻略心得、解謎線索、收集品狀態與多人合作資訊。這些內容可能分散在手機備忘錄、截圖、社群貼文、試算表、聊天紀錄或紙本筆記中，導致查找困難、資訊重複、版本混亂，也不容易在不同遊戲之間建立一致的整理方式。

本專案希望設計一個以「遊戲筆記」為核心概念的網站，讓玩家可以用自己喜歡的方式去保存遊戲中的知識與進度。網站不只是單純的筆記工具，而是針對遊戲情境設計的個人資料庫，協助玩家在長時間遊玩、暫停後回鍋、攻略複雜系統或分享心得時，能快速找回重要資訊。

## 2. 專案目標

### 2.1 核心目標

- 提供玩家一個集中管理遊戲筆記的平台。
- 讓使用者先選擇遊戲，再進入該遊戲專屬的筆記與管理功能。
- 支援依照遊戲建立獨立筆記空間，讓每款遊戲的內容分開管理。
- 降低玩家回顧進度與查找資訊的成本。
- 讓筆記內容保有自由書寫與快速回顧的彈性。
- 建立適合長期累積、持續補充與反覆查閱的遊戲知識空間。

### 2.2 使用者價值

- 玩家可以快速知道自己上次玩到哪裡。
- 玩家可以保存攻略心得、配裝思路與遊戲內發現。
- 玩家可以追蹤收集品、任務、成就或素材取得狀態。
- 玩家可以依遊戲建立獨立筆記空間，避免不同遊戲資訊混雜。
- 玩家可以透過遊戲名稱與平台搜尋快速找到過去記錄。

### 2.3 設計方向

- 介面應以「先選遊戲，再整理該遊戲內容」為主要流程。
- 首頁應優先呈現使用者的遊戲清單，而不是直接混合所有筆記。
- 資訊呈現需清楚、安靜、耐看，適合長時間閱讀與編輯。
- 操作流程應簡單直接，讓玩家能在遊戲間隙快速記錄。
- 功能優先支援實用性，再逐步擴充社群、分享或進階分析能力。

## 3. 名詞定義

| 名詞 | 定義 |
| --- | --- |
| 使用者 | 註冊或使用本網站建立遊戲筆記的人。 |
| 遊戲筆記 | 使用者針對某款遊戲建立的文字、圖片、清單、表格或其他記錄內容。 |
| 遊戲庫 | 使用者所有遊戲檔案的集合，也是網站首頁的主要內容。 |
| 遊戲檔案 | 一款遊戲在網站中的獨立資料空間，可包含多篇筆記、紀念圖片與進度資訊。 |
| 筆記條目 | 單篇筆記內容，例如攻略心得、任務記錄、角色配置、地圖線索等。 |
| 任務追蹤 | 用於記錄遊戲任務、支線、目標或待辦事項完成狀態的功能。 |
| 進度摘要 | 顯示某款遊戲目前狀態的簡短資訊，例如遊玩進度、近期目標、待完成事項。 |
| 收藏項目 | 使用者標記為重要、常用或稍後查閱的筆記或資料。 |
| 範本 | 預先設計好的筆記格式，協助使用者快速建立常見類型的遊戲筆記。 |

## 4. 功能需求

### 4.1 遊戲檔案管理

使用者可以建立、編輯與刪除遊戲檔案。每個遊戲檔案應可設定遊戲名稱、封面圖片、平台、遊戲類型、遊玩狀態與簡短描述。

基本欄位包含：

- 遊戲名稱
- 遊戲封面
- 遊戲平台、官方連結
- 遊玩狀態：未入坑、遊玩中、半退坑、已退坑
- 簡短備註

### 4.2 筆記建立與編輯

使用者可以在特定遊戲檔案下建立筆記條目。筆記應支援標題、正文與更新時間。

筆記編輯需支援：

- 新增筆記
- 編輯筆記
- 刪除筆記
- 儲存草稿
- 點擊筆記放大閱讀
- 標記為收藏

### 4.3 遊戲紀念圖片

網站需提供遊戲紀念圖片功能，讓使用者保存遊戲截圖或值得紀念的畫面。

圖片功能需支援：

- 上傳圖片
- 自訂圖片名稱
- 未輸入名稱時使用檔名
- 圖片牆顯示
- 點擊圖片放大預覽
- 刪除圖片

### 4.4 搜尋與篩選

使用者可以搜尋自己的遊戲筆記，並透過條件縮小結果。

搜尋與篩選條件包含：

- 關鍵字
- 遊戲名稱
- 收藏狀態
- 建立時間
- 更新時間

搜尋結果應清楚顯示遊戲名稱、平台、遊玩狀態與最後更新時間。

### 4.5 任務與待辦追蹤

使用者可以針對每款遊戲建立任務或待辦清單，用於追蹤尚未完成的目標。

任務項目需支援：

- 任務名稱
- 任務描述
- 完成狀態
- 優先順序
- 關聯筆記
- 關聯地點或角色
- 備註

任務狀態可包含：

- 未開始
- 進行中
- 已完成
- 暫時擱置

### 4.6 進度摘要

每個遊戲檔案應有一個進度摘要區，讓使用者快速回顧目前狀態。

進度摘要可包含：

- 最近更新的筆記
- 未完成任務數量
- 收藏筆記
- 下一步目標
- 最近遊玩日期
- 使用者自訂提醒

### 4.7 筆記範本

網站可提供常見遊戲筆記範本，幫助使用者快速開始記錄。

範本例子包含：

- 任務攻略筆記
- 角色養成筆記
- 裝備配置筆記
- 地圖探索筆記
- 素材收集筆記
- Boss 攻略筆記
- 解謎線索筆記

### 4.8 圖片與截圖管理

使用者可以在筆記中加入遊戲截圖，用於記錄地圖位置、道具資訊、角色配置或重要畫面。

圖片功能需支援：

- 上傳圖片
- 圖片預覽
- 圖片說明文字
- 在筆記中插入圖片
- 刪除圖片

### 4.9 收藏與快速存取

使用者可以收藏重要筆記，並在首頁或遊戲檔案頁快速開啟。

適合收藏的內容包含：

- 常用攻略
- 角色配裝
- 素材路線
- 重要地圖位置
- 尚未解完的謎題線索

### 4.10 遊戲選擇首頁

網站首頁應作為「遊戲選擇頁」，讓使用者先從自己的遊戲庫中選擇一款遊戲。初次使用時，首頁不應預先顯示任何範例遊戲、範例筆記、範例任務或假資料，只呈現空狀態與新增第一款遊戲的入口。使用者建立並選擇某款遊戲後，才顯示該遊戲的筆記、紀念圖片與進度摘要。

首頁內容可包含：

- 新增遊戲檔案入口
- 遊戲名稱搜尋
- 依平台、遊玩狀態或遊戲類型篩選
- 每款遊戲的進度摘要，例如未完成任務數量、最近更新時間、收藏筆記數量

當使用者尚未建立任何遊戲檔案時，首頁應顯示：

- 空狀態標題，例如「還沒有遊戲筆記」。
- 簡短說明，提醒使用者先新增一款遊戲。
- 主要操作按鈕，例如「新增第一款遊戲」。
- 可選的輔助操作，例如匯入資料或查看建立範本。
- 搜尋、篩選、最近開啟、收藏與統計區塊應保持空白或停用，不顯示假資料。

使用者選擇遊戲後，系統應進入該遊戲的專屬管理頁。該頁才提供筆記建立、圖片紀念與遊戲資料編輯等功能。

### 4.11 響應式設計

網站需支援桌面與手機瀏覽。手機版應讓使用者能快速查看筆記、搜尋內容與新增簡短記錄。

基本要求包含：

- 桌面版適合瀏覽大量資訊與編輯長篇筆記。
- 手機版適合快速查找、補充與檢視。
- 主要操作按鈕需易於點擊。
- 文字閱讀需清楚，不應過度擁擠。

## 5. 後續可延伸功能

以下功能不一定列入第一版，但可作為後續擴充方向：

- 筆記分享連結
- 公開攻略頁
- 遊戲資料庫串接
- 成就或收集品進度圖表
- 離線瀏覽
- 匯出 Markdown、PDF 或純文字
- 跨裝置同步
- AI 協助整理筆記或產生摘要
